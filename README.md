# cct-scripts

A collection of [CC:Tweaked](https://tweaked.cc/) scripts for Minecraft modpacks.
Scripts are deployed directly to Minecraft computers over SSH using `deploy.sh`.

## Scripts

| Script | Location | Description |
| ------ | -------- | ----------- |
| `material_usage_monitor.lua` | `src/resource-ticker/` | Live AE2 network dashboard: top movers, curated watchlist, depletion alarm with speaker, touch buttons, state persists across restarts. |
| `usage_config.lua` | `src/resource-ticker/` | CLI companion to the monitor. Manage the watchlist and auto-scan settings with guided, validated prompts. |
| `autocraft_monitor.lua` | `src/` | Crafting CPU dashboard. Shows BUSY/IDLE status, current job, and a recent-activity log, polling every 2s. |
| `todos.lua` | `src/` | Persistent to-do list on an Advanced Monitor. Touch to toggle done/undone, tap to delete, speaker feedback. |

## Deploying

Copy your credentials into `.env` (see `.env.example`), then run:

```bash
# Push both resource-ticker scripts to computer 9
./deploy.sh --computer 9 \
  src/resource-ticker/material_usage_monitor.lua \
  src/resource-ticker/usage_config.lua

# Push to multiple computers in one shot
./deploy.sh --computer 9 --computer 12 \
  src/resource-ticker/material_usage_monitor.lua
```

The script reads the world name automatically from `server.properties` on the
remote host, so you only need to set `SSH_HOST` and `SERVER_DIR` in `.env`.
See `deploy.sh` header for full flag reference.

## Server permission setup (one-time)

By default, writing to the ComputerCraft save directory requires `sudo su - minecraft`.
To allow passwordless SCP deploys, run the following once as a sudoer on the server:

```bash
sudo usermod -aG minecraft kevin
sudo chmod -R g+rwX /path/to/server/world/computercraft/computer
sudo find /path/to/server/world/computercraft/computer -type d -exec chmod g+s {} \;
```

Log out and back in for the group change to take effect, then verify:

```bash
touch /path/to/server/world/computercraft/computer/9/test_write
rm    /path/to/server/world/computercraft/computer/9/test_write
```

For fully passwordless deploys, also set up key-based SSH auth:

```bash
ssh-keygen -t ed25519 -C "deploy-key"
ssh-copy-id user@hostname
```
